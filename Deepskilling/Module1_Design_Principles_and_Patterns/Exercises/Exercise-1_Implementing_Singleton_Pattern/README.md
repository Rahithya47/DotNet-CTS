# Exercise 1 - Implementing Singleton Pattern

## Objective

Implement the Singleton Design Pattern to ensure that only one instance of a Logger class is created throughout the application's lifecycle.

## Design Pattern

**Singleton Pattern**

The Singleton Pattern restricts the instantiation of a class to one object and provides a global point of access to it.

## Files

- Logger.java
- SingletonTest.java

## Output

```
Logger Instance Created.
[LOG] Application Started
[LOG] User Logged In
Both objects are the same instance.
```

## Advantages

- Ensures only one instance exists.
- Saves memory by avoiding unnecessary object creation.
- Provides a global access point.
- Commonly used for logging, configuration, and database connections.

## Real-World Applications

- Logging Frameworks
- Database Connection Manager
- Configuration Manager
- Cache Manager

- Java
